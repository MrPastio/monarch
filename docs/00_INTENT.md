# Замысел Monarch

Monarch - локальная архитектура для AI-управляемой персональной экосистемы.

Для пользователя это может выглядеть как один ассистент, но внутри это федерация систем:

- стабильное ядро
- сеть роутеров
- типизированные инструменты
- локальные модельные рантаймы
- память и профили
- доменные модули
- UI и голосовые интерфейсы
- безопасность и permissions

Цель не в том, чтобы сделать ассистента с большим количеством кнопок. Цель - создать операционный слой, где отдельные системы можно подключать, осмыслять, автоматизировать и контролировать через AI-командование.

## Продуктовый Принцип

ИИ-агент отдает умные приказы. Доменные модули выполняют реальную работу.

Пример Security Center:

- Агент понимает намерение пользователя.
- Security модуль владеет сканами, integrity-проверками, audit tail, baseline actions и safety rules.
- Ядро проверяет permissions и маршрутизирует команды.
- Выполнение идет через типизированные capabilities, а не через произвольные сырые скрипты.

## Local-First Обещание

Monarch по умолчанию должен работать локально:

- локальные настройки
- локальная память
- локальные логи
- локальные модели там, где это возможно
- явное включение сетевых провайдеров
- никаких plaintext-секретов в tracked-файлах проекта
